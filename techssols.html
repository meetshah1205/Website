Now let's talk about the most famous ways that hackers use to infiltrate your devices and apps.

One of the easiest and most effective methods of database hacking is SQL (Structured Query Language) Injections.

What is SQL?

SQL is a programming language widely used to create databases to store data for a website, app, etc.

You can read this Wikipedia article about SQLto know more about SQL and SQL Injections.

But here is a quick explanation about SQL:




Structured Query Language (SQL):

SQL stands for Structured Query Language.

It is a domain-specific language used to manage and manipulate relational databases. SQL is used to perform tasks such as querying, updating, inserting, and deleting data in a database.

SQL Commands:

  • SELECT: Retrieves data from one or more tables.


  • INSERT: Adds new records to a table.


  • UPDATE: Modifies existing records in a table.


  • DELETE: Removes records from a table.


Database Manipulation:

  • CREATE DATABASE: Creates a new database.
  • USE DATABASE: Specifies the database to be used.
  • DROP DATABASE: Deletes an existing database.

Table Manipulation:

  • CREATE TABLE: Creates a new table with specified columns and data types.


  • ALTER TABLE: Modifies an existing table (e.g., adds or removes columns).


  • DROP TABLE: Deletes an existing table and its data.


Data Types:

  • INT: Integer (whole numbers).


  • VARCHAR: Variable-length character strings.


  • DATE: Represents date values.


  • BOOLEAN: Represents true or false values.


Query Filtering:

  • WHERE: Filters data based on specified conditions.


  • ORDER BY: Sorts the result set in ascending or descending order.


  • GROUP BY: Groups rows based on a specified column.


Data Retrieval:

  • JOIN: Combines rows from two or more tables based on a related column.


  • INNER JOIN, LEFT JOIN, RIGHT JOIN, FULL JOIN: Different types of joins.


  • SUBSELECT (or subquery): Nesting a query within another query.


Database Security:

  • GRANT: Provides specific privileges to users or roles.


  • REVOKE: Removes specific privileges from users or roles.






Now that you have a good idea of what SQL is and how does it work, Let's now talk about SQL Injections.



SQL Injections are used by Hackers to hack into someone's online account without their password(sometimes not even username or email). What's scary about them is that Hackers don't need any fancy tools to hack into somebody's account using SQL Injections.



Let's first understand how SQL Injections function so you can understand them and defend yourself against them.



When you enter your Username and password into a website it connects to a data-base and validates your id and password. If the id and password you entered "BOTH" exist on the database on the same place in the databse then you login. It runs a search in SQL to see weather they exist or not. The search query in SQL should look something like this

SELECT * FROM users WHERE username = 'admin' AND password = ' s3cret123'

There are several methods of SQL injections:


1. "OR" Statements:

If you know even the basics of English, you might know that "OR" means if a condition does not apply, then do something else.

Example:
The weather is good "OR" bad.

So as you saw in the SQL query showed earlier that it stores the username and password in quotes( 'Username',   'Password' )

This is how Hackers get into an account using "OR" Statements to login to your account.

[USERNAME]' OR 1<2



Explanation:

The [USERNAME] as you might have guessed stands for the username of the account and there is a single-quote(') after it because in the SQL Query showed earlier the Username was stored inside quotes so that single-quote(') breaks that such that the content after that is considered a SQL Query, so the OR 1>2 is ofcource true so it lets you login. This technique is very commonly used by Hackers to infiltrate accounts.

Because there is one thing you always have to remember:
"No matter how good of a password you have, there will be a Hacker that will easily infiltrate your account with absolute ease using SQL Injections or any other method that you may never know......."

                                                                                                           -Somebody should have said this.

Here is how SQL sees this:

SELECT * FROM users WHERE username = 'admin'  OR 1<2' AND password = ' s3cret123'

So it basically just tells SQL to login if the username is admin or the condition 1<2 is true just continue.

2. Comments

Comments are things (or text) to be ignored by the compiler or interpreter (Programs that execute the code and show error if the code is wrong).

Comments in any Programming language are for humans to read when maintaining or reading the code.

For example comments in Java or JavaScript are write by putting double-forward-slashes (//) before the line.

Example:

if (0<1){
   console.log("Its less than 1")
// This executes when a is less than 1.
}




By the way you don't need to learn Java or JavaScript to understand SQL injections.

But this was just fro your information.

Anyway, let's continue.

As you saw in that JavaScript code that there was a comment after console.log("Its less than 1").

That comment will be ignored by JavaScript when you run the program.

Comments in SQL are written by putting double-dashes (--) after the line.

So Hackers can do something like this to hack into your account using SQL comments:

[USERNAME]' --

Explanation:

All the things after double-dashes (--) will be ignored.

SQL sees this like this:

SELECT * FROM users WHERE username = 'admin'   -- ' AND password = 's3cret123'



The grey text will be ignored and the Hacker will login.

Hackers use this technique a lot when using SQL Injections.





Now lets talk about how can you protect yourself and your devices against SQL injections.



  • Don't use SQL

    • This seems to be the most obvious choice but its not.

    Not using SQL as your data-base is not seen very commonly as SQL is the most popular and widely used language for making data-bases.

    If you don't use SQL you will miss out on so much but if its more than your users privacy and security of their account and it might even be that some Hacker is smart enough to guess the usernames of your account and misuse it in many ways, then go for it nobody is there to stop you.

    To use any of the methods of SQL Injections, the hacker will first ahve to figure out if the website on which his target has an account on, even uses SQL. This process is quite time consuming.



  • Try trickey usernames

    • Any Hacker will go for the most obvious usernames first.

    So you can use trickey usernames to have some extra security.



  • Hope

    • If you don't want to do any of the above listed things, you can just pray and hope nobody Hacks your account.

  • Use Two or Three step verification.


    • This is the method that big companies like Google use to protect their users privacy.

    When a user (or Hacker) tries to login to an account you send a verification to their mobile where if they agree login on the other device they press yes, if they don't agree they should press No.

    Or use three step verification where they enter password (avoidable easily with SQL Injections as we discussed earlier), then press Yes on their phone and enter a 6 digit code sent on their email.

    A Hacker would most likely fail to login.

    If the Hacker manages to login even after all that, ask them for their password in every nessecery action (like Linux and its distros ask passwords for every download from their software store, every time you login, every app update, every sudo command.)
